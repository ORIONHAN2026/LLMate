# Framework 重构计划

## 目标
- 删除 Anthropic/Gemini provider，统一使用 OpenAI 协议
- 按 workMode（conversation/contract/invoice/chatroom）创建独立接口类
- 简化 BaseLlmProvider 为 OpenAI 专用

## 新文件结构

```
lib/framework/
├── llm_framework.dart              # barrel export（精简）
├── llm_client.dart                 # LlmClient（从 llm_hub.dart 拆出）
├── openai_provider.dart            # OpenAI HTTP/SSE 网络层
├── modes/
│   ├── work_mode_strategy.dart     # 抽象接口：消息/提示词/工具组装
│   ├── conversation_mode.dart      # 对话模式
│   ├── contract_mode.dart          # 合同模式
│   ├── invoice_mode.dart           # 发票模式
│   └── chatroom_mode.dart          # 聊天室模式
├── common/
│   ├── message_builder.dart        # 消息构建工具（保留）
│   └── system_prompts.dart         # 公共提示词常量（保留）
```

## 架构设计

### 1. WorkModeStrategy（抽象接口）

```dart
abstract class WorkModeStrategy {
  /// 模式名称
  String get modeName;
  
  /// 构建模式特有的系统提示词消息列表
  Future<List<Map<String, dynamic>>> buildSystemMessages({
    ChatModel? model,
    ChatSession? session,
  });
  
  /// 构建该模式下可用的工具列表
  List<Map<String, dynamic>> buildTools(ChatSession? session);
}
```

### 2. 四个模式实现类

每个类实现 `WorkModeStrategy`，负责：

**ConversationMode（对话模式）**
- 系统提示词：模型自定义 prompt + 技能 prompt + 深度思考 + 工作目录 + 核心规则 + 语言
- 工具：基础系统工具（node/python/ocr/file） + MCP + Skill

**ContractMode（合同模式）**
- 系统提示词：同上 + `CommonSystemPrompts.contractMode()` 合同专用流程
- 工具：合同专用工具（contract_inspect/content_update/process_update/disguss_update）+ 基础工具 + MCP + Skill

**InvoiceMode（发票模式）**
- 系统提示词：同上 + `CommonSystemPrompts.invoiceMode()` 发票专用流程
- 工具：发票专用工具（invoice_summary/detail_update/reimbursement_update）+ 基础工具 + MCP + Skill

**ChatroomMode（聊天室模式）**
- 系统提示词：同上 + `CommonSystemPrompts.chatroomMode()` 角色加载
- 工具：角色管理工具（role_create/update/delete）+ 基础工具 + MCP + Skill

### 3. OpenAiProvider（网络层）

只负责 HTTP 通信，删除协议无关逻辑：
- `sendMessageStream()` - SSE 流式请求
- `sendMessageStreamWithMessages()` - 预构建消息流式请求
- `sendMessage()` - 非流式请求
- `buildRequestData()` - 构建 OpenAI 格式请求体
- `buildAuthHeaders()` - 认证头
- `validateConfiguration()` - 配置验证
- SSE 解析（使用 OpenAIResponse 模型）
- 错误处理

### 4. LlmClient（简化）

- 删除 `LlmHub`（不再需要 provider 注册中心）
- 直接创建 `OpenAiProvider`
- 根据 `session.workMode` 创建对应 `WorkModeStrategy`
- 调用 strategy 的方法组装消息/工具，调用 provider 发送请求

## 实施步骤

### Step 1: 创建 modes/ 目录和策略接口
- 创建 `work_mode_strategy.dart` 抽象接口
- 创建 `conversation_mode.dart`（从 base_provider.dart 的 buildMessages/buildTools 提取对话模式逻辑）
- 创建 `contract_mode.dart`
- 创建 `invoice_mode.dart`
- 创建 `chatroom_mode.dart`

### Step 2: 重构 OpenAiProvider
- 将 BaseLlmProvider 中的 OpenAI 相关逻辑合并到 OpenAiProvider
- 保留 network 层（Dio + SSE 解析）
- 保留 buildRequestData / buildAuthHeaders / validateConfiguration
- 删除 protocol 路由相关代码
- 将 `_safeNameToOriginal` 等工具名映射移入

### Step 3: 简化 LlmClient
- 从 llm_hub.dart 中提取 LlmClient 到 llm_client.dart
- 删除 LlmHub 注册中心
- LlmClient 直接创建 OpenAiProvider + WorkModeStrategy
- buildMessages / buildTools 调用 strategy 的对应方法

### Step 4: 更新 barrel export
- 更新 `llm_framework.dart` 导出新文件

### Step 5: 修改外部调用方
- 更新 import 路径
- 将 `LlmHub.createProvider()` 替换为直接创建 OpenAiProvider
- 将 `BaseLlmProvider.resolveOriginalToolName()` 替换为 `OpenAiProvider.resolveOriginalToolName()`

### Step 6: 删除旧文件
- 删除 `anthropic_provider.dart`
- 删除 `gemini_provider.dart`
- 删除 `base_provider.dart`
- 删除 `llm_hub.dart`

2. 确认所有 import 路径正确
3. 确认 LlmClient 在各处的调用不受影响
4. 确认 workMode 切换时使用正确的 strategy
