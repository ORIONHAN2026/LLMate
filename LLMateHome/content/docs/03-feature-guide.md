# 功能说明

本页汇总 LLMate 当前已实现的全部功能，帮助你快速建立整体认知。每个功能都标注了在应用中的入口位置，方便对照操作。

---

## 一、多模型统一管理

在 **模型管理** 中集中接入和管理各类大模型，支持为不同模型分别配置 API 地址、密钥、温度、最大 Token 等参数。

- **已内置预设的供应商**：OpenAI、Anthropic Claude、Google Gemini、DeepSeek、阿里云百炼（通义千问）、智谱 AI、Ollama（本地）、腾讯云 TokenHub、Kimi（Moonshot）、小米 Mimo
- **自定义接入**：若列表中没有你的供应商，可选择「自定义」并填写 Base URL 与协议（OpenAI / Anthropic / Gemini 兼容格式）
- **在线 / 本地模型**：既可接入云端 API，也可用 Ollama 在本地运行开源模型（地址默认 `127.0.0.1:11434`）
- 对话时通过模型选择器一键切换，无需重启会话

> 详见 [大模型配置](/docs/03-model-config/)。

---

## 二、工作模式

LLMate 针对不同业务场景提供专属工作模式，右侧导航栏会随模式自动切换为对应的结构化面板：

| 模式 | 右侧面板 | 典型用途 |
|------|---------|---------|
| 合同 | 合约要点 · 合同履约 · 合同争议 · 备忘录 | 合同审查、要点提取、履约跟踪 |
| 发票 | 发票汇总 · 发票明细 · 报销记录 · 备忘录 | 发票识别、报销统计 |
| 聊天室 | 角色列表 · 备忘录 | 多角色群聊模拟、多角度讨论 |
| 创作 | 灵感 · 脑图 · 草稿 | 头脑风暴、思维导图、草稿管理 |

此外，新建会话时还可选择 **商务 / 财务 / 法务 / 市场** 等场景标签，用于区分会话的业务归属。

> 详见 [工作模式](/docs/04-workmodes/)。

## 三、MCP 服务编排

LLMate 原生支持 **Model Context Protocol (MCP)**，让 AI 连接外部工具与数据源（文件系统、数据库、内部 API、搜索引擎等）。

- **连接协议**：Stdio（本地进程）、SSE（远程服务）、HTTP、Streamable HTTP
- **添加方式**：在 **MCP 管理** 中粘贴标准 `mcpServers` JSON 或填写命令/URL 即可添加
- **绑定使用**：在会话设置中勾选需要启用的 MCP 服务，AI 即可在该会话内调用对应工具
- 会话级与模型级 MCP 会自动合并、去重

> 详见 [增加 MCP](/docs/07-add-mcp/) 与 [配置 MCP](/docs/08-mcp-config/)。

---

## 四、用量统计与配额管理

每个会话实时统计 Token 消耗与费用，并支持设置配额上限，防止资源被滥用：

- **统计项**：输入 / 输出 Token、总 Token、累计费用、请求次数
- **配额项**：Token 上限、费用预算上限、请求次数上限、重置周期（每日 / 每月 / 不重置）
- **超额拦截**：达到配额后自动阻断后续请求，管理员可手动重置
- **可视化**：支持按分 / 时 / 天 / 月 / 年粒度的用量曲线图

> 详见 [会话用量管理](/docs/06-usage-management/)。

---

## 五、本地 HTTP / HTTPS 代理服务

LLMate 内置本地代理，把配置好的模型接口透传为 **OpenAI 兼容 API**，方便企业内部系统直接调用：

- 支持配置 HTTP / HTTPS 监听端口（默认 80 / 443）
- 支持 API Key 认证、外部访问开关、配额与审计中间件
- 每个会话都有独立的服务地址，可在会话设置中复制分发

> 详见 [会话服务地址](/docs/05-session-service/)。

---

## 六、系统提示词与免授权访问

- **企业级系统提示词**：可为会话单独设置系统提示词，统一规范 AI 输出行为
- **免授权模式**：开启后使用者无需配置密钥即可使用对应会话，降低推广门槛（注意仅在可信内网环境启用，详见 [安全说明](/docs/10-security/)）

---

## 七、国际化与主题

- **8 种界面语言**：中文、English、日本語、ไทย、Tiếng Việt、한국어、Français、Deutsch
- **主题**：支持跟随系统 / 深色 / 浅色，可在设置中切换

---

## 八、会话与组织管理

- 会话是核心管理单元，每个会话可独立绑定模型、密钥、系统提示词、工作目录、MCP 与配额
- 支持分组管理，便于按团队 / 项目归类
- 支持会话复制、重命名、导出与禁用

> 详见 [创建会话](/docs/04-create-session/)。
