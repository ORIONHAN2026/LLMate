# 场景案例

下面通过几个典型场景，演示如何用 LLMate 把软件真正用起来。所有案例都基于已实现的真实功能。

---

## 场景一：合同审查与履约跟踪

**适用模式**：合同模式

1. 新建会话，选择 **合同** 模式，绑定一个具备长上下文能力的模型（如 DeepSeek、Claude）
2. 在会话设置中设置 **工作目录**（存放合同文件的目录）
3. 对话中让 AI 解析工作目录里的合同文件，AI 会自动提取：
   - 合约要点（缔约方、金额、期限、支付条款、违约责任）
   - 合同履约进度
   - 合同争议点
4. 右侧「合约要点」「合同履约」「合同争议」面板会实时生成结构化卡片，可在「备忘录」中补充备注

> 适合法务、商务、采购等岗位对批量合同做初审与归档。

---

## 场景二：发票识别与报销统计

**适用模式**：发票模式

1. 新建会话，选择 **发票** 模式
2. 将发票图片 / PDF 放入工作目录，或直接发送给 AI
3. AI 识别发票信息后，右侧「发票汇总 / 发票明细 / 报销记录」面板自动统计金额、分类与报销状态
4. 导出汇总结果用于财务对接

> 适合财务、行政岗位快速做报销整理。

---

## 场景三：用 MCP 接入企业知识库做问答

**适用能力**：MCP + 本地 HTTP 代理

1. 在 **MCP 管理** 中添加一个知识库 / 数据库 MCP 服务（SSE 或 Streamable HTTP 方式）
   ```json
   {
     "mcpServers": {
       "company-kb": {
         "url": "https://kb.internal.company.com/mcp"
       }
     }
   }
   ```
2. 在目标会话的 MCP 设置中勾选该服务并保存
3. 在对话中直接提问「公司差旅报销标准是什么？」，AI 会调用 MCP 工具检索知识库后回答
4. 如需让其他内部系统也能用，可在 **会话服务地址** 中开启本地 HTTP 代理并复制地址给其他系统调用

---

## 场景四：多角色头脑风暴（聊天室模式）

**适用模式**：聊天室模式

1. 新建会话，选择 **聊天室** 模式
2. 在「角色列表」中定义多个虚拟角色（如「产品经理」「工程师」「设计师」）
3. 提出一个议题，AI 依次以不同角色视角展开讨论，多角度输出写在「备忘录」中留档

> 适合产品评审、方案讨论、培训演练。

---

## 场景五：内容创作（创作模式）

**适用模式**：创作模式

1. 新建会话，选择 **创作** 模式
2. 在「灵感」面板收集素材，在「脑图」面板梳理结构，在「草稿」面板迭代成文
3. 结合系统提示词约束输出风格，保证内容符合企业调性

---

## 场景六：作为团队统一 AI 网关

**适用能力**：本地 HTTP 代理 + 配额 + 审计

1. 管理员在 LLMate 中配置好各供应商模型与 API Key
2. 为每个部门 / 项目创建独立会话，设置 **用量配额** 与 **免授权模式**
3. 开启本地 HTTP 代理，把每个会话的 OpenAI 兼容地址分发给对应团队
4. 在用量面板查看各部门消耗，异常用量可通过审计日志追溯

> 这样团队无需各自申请密钥，统一由 LLMate 做代理、计量与管控。
