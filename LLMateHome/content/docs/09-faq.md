# 常见问题

---

### LLMate 支持哪些模型平台？

已内置预设：OpenAI、Anthropic Claude、Google Gemini、DeepSeek、阿里云百炼（通义千问）、智谱 AI、Ollama（本地）、腾讯云 TokenHub、Kimi（Moonshot）、小米 Mimo。未列出的供应商可选择「自定义」并填写 Base URL 与协议（OpenAI / Anthropic / Gemini 兼容格式）接入。

---

### 添加模型后无法对话，提示 401 / 连接超时？

- **401**：API Key 错误或已失效，请到 **模型管理** 重新填写
- **连接超时**：检查 Base URL 是否正确、网络是否可访问目标服务商；使用 Ollama 时确认本地服务已启动且地址为 `127.0.0.1:11434`
- 部分服务商需要特定的请求头或组织 ID，可在模型设置的高级项中补充

---

### 本地 HTTP 服务提示端口被占用？

默认 HTTP 端口 80、HTTPS 端口 443。若被其他程序占用，到设置 **本地 HTTP 服务** 中修改为其他端口（如 `8080`）并重启服务。修改后需重启 LLMate 才能生效。

---

### MCP 连接失败怎么办？

- **Stdio 方式**：确认命令与参数正确，且运行环境已安装所需依赖（如 `npx`）。建议使用绝对路径，避免路径注入
- **SSE / Streamable HTTP 方式**：确认服务 URL 可达、认证信息正确。LLMate 会在 SSE 与 Streamable HTTP 之间自动回退尝试
- 可在 MCP 管理中查看该服务的工具列表是否正常加载，工具为空通常是连接未成功

---

### 支持哪些界面语言？如何切换？

支持 **中文、English、日本語、ไทย、Tiếng Việt、한국어、Français、Deutsch** 共 8 种。在 **设置 → 语言** 中选择，重启或即时生效（取决于系统）。

---

### 什么是「免授权模式」，安全吗？

开启后，使用者在打开该会话时**无需配置密钥**即可直接对话，LLMate 会使用会话绑定的密钥代理请求。它降低了使用门槛，但意味着任何能打开该会话的人都能消耗配额，**请仅在可信内网环境开启**，并配合配额与审计使用。详见 [安全说明](/docs/10-security/)。

---

### 配额用完了怎么办？

达到 Token / 费用 / 请求次数上限后，LLMate 会拦截后续请求。管理员可在会话设置的配额区手动 **重置**，或等待重置周期（每日 / 每月）自动归零。

---

### 我的对话和数据存在哪里？会传给别人吗？

所有会话、消息、用量与审计数据**默认只存储在本地**（位于用户目录下的 `.llmate` 目录），不会上传到 LLMate 官方服务器。模型请求只会发往你在模型中配置的供应商地址。详见 [安全说明](/docs/10-security/)。

---

### 如何切换深色 / 浅色主题？

在 **设置 → 外观** 中选择「跟随系统 / 深色 / 浅色」。

---

### 支持手机（Android / iOS）吗？

当前 LLMate 仅提供 **桌面端（macOS / Windows / Linux）与 Web** 版本，不包含 Android / iOS。移动端请使用桌面版或 Web 版访问。

---

### 想自己改功能 / 二次开发，哪里找源码？

LLMate 以 GPL v3.0 开源，可参考 [源码编译安装](/docs/02-source-build/) 自行构建与定制。
