# 安全说明

LLMate 的定位是「企业 AI 安全治理」工作台。本页说明数据的存储与流转边界，以及使用时的安全建议。

---

## 一、数据存哪里、流向哪

- **本地优先**：所有会话、消息、用量统计、审计记录默认存储在用户目录下的 `.llmate` 目录（如 `~/.llmate/`），**不会上传到 LLMate 官方服务器**
- **模型请求流向**：你与大模型对话时，请求只发往你在「模型管理」中配置的供应商 API 地址（如 OpenAI、DeepSeek 等），密钥也仅保存在本地配置中
- **审计数据库**：审计日志写入本地 `audit.db`，记录请求来源、调用模型、Token 消耗等，便于事后追溯

> 结论：只要你不主动把数据导出或接入外部 MCP，数据就留在你自己的机器上。

---

## 二、密钥与认证

- **API Key 仅本地保存**：模型密钥、会话密钥都存储在本地，不会外泄给 LLMate
- **本地 HTTP 代理认证**：开启本地 HTTP 服务后，建议启用 **API Key 认证**，避免局域网内未授权调用
- **配额防护**：代理层内置配额守卫（quota guard），超额自动拦截，防止密钥被滥用

---

## 三、MCP 权限隔离

- MCP 服务**按会话绑定**，未勾选的会话无法调用对应工具，避免跨部门越权访问
- 涉及敏感资源（文件系统、数据库、内部 API）的 MCP 建议只绑定到特定岗位的会话
- Stdio 方式请使用命令的绝对路径，避免路径注入风险
- 所有 MCP 工具调用会经过审计中间件记录

---

## 四、免授权模式的使用边界

「免授权模式」让使用者无需密钥即可对话，本质是让 LLMate 用会话绑定的密钥代为请求。它**降低了门槛也放大了风险**：

- 仅在可信内网环境开启
- 务必配合用量配额与审计日志使用
- 对外网暴露的代理服务不要开启免授权

---

## 五、传输加密（HTTPS）

本地 HTTP 服务支持启用 **HTTPS**，可加载证书实现加密传输。对外提供服务时建议开启 HTTPS，避免密钥与对话内容在链路上明文传输。

---

## 六、内容合规

- 可为企业 / 会话设置 **系统提示词**，从源头约束 AI 输出，降低违规内容风险
- 审计日志可回溯每条请求的发送内容与消耗，满足合规留痕需求

---
