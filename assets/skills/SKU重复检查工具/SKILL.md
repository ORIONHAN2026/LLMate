# SKU 重复检查（一品多码检测）

## Overview

对数据总表进行全量 SKU 重复检查。数据总表是一个 Excel (.xlsx) 文件，包含两个 Sheet：

- 第一个Sheet（商品原始表）：已存在的原始商品数据，固定表头 `商品编码 | 自定义名称 | 品牌`
- 第二个Sheet（动态商品池）：每天新增的商品数据，固定表头 `名称 | 品牌 | 编码 | 单位`

判定规则（一品多码检测，支持名称模糊匹配）：

匹配分两步：
1. **品牌精确匹配**：两个商品的品牌必须完全一致
2. **名称子串包含匹配**：标准化后（去空格/括号/转小写）一个名称完全包含另一个即判定为同一商品

| 品牌 | 名称 | 编码 | 结果 |
|------|------|------|------|
| 相同 | 匹配（精确/子串） | 不同 | 🔴 **一品多码，标红** |
| 相同 | 匹配（精确/子串） | 相同 | ✅ 同一商品，不标红 |
| — | 不匹配 | — | ✅ 不同商品，不标红 |

名称匹配示例：
- `石墨盘根 10*10mm` ↔ `森耐德(sennaide) 石墨盘根 10*10mm 5kg/卷` → ✅ 匹配（子串包含）
- `HP 硒鼓 CF280A` ↔ `惠普（HP）硒鼓 CF280A/80a 黑色` → ✅ 匹配（子串包含，品牌另单独比对）
- `张三的猫咪` ↔ `一只猫咪张三家的` → ✅ 匹配（核心词重叠，子串包含）

## Workflow

### Step 1: 确认数据总表路径

开始执行前，必须向用户确认数据总表（.xlsx 文件）的路径。如果用户未提供路径，主动询问：

「请提供数据总表（.xlsx文件）的路径，该文件应包含两个Sheet：商品原始表和动态商品池。」

不要猜测路径，不要在未拿到明确路径前执行。

### Step 2: 执行重复检查

使用 `scripts/check_duplicates.py` 脚本执行检查。**直接对原文件操作，无需备份、无需复制到工作区**：

```bash
python3 scripts/check_duplicates.py "FILE_PATH"
```

脚本会自动完成以下工作：

1. 验证文件存在并能正常打开
2. 自动识别两个 Sheet（默认第1个=商品原始表，第2个=动态商品池）
3. 按固定表头定位列（表头必须匹配：Sheet1=商品编码/自定义名称/品牌，Sheet2=名称/品牌/编码/单位）
4. 按品牌分组原始表，遍历动态商品池逐一比对（品牌精确 + 名称子串包含）
5. 发现一品多码的行，以 **红色底色 + 白色字体** 标红该行
6. 在原文件上直接保存修改
7. 输出 JSON 格式的检查报告

脚本输出为 JSON，关键字段：

- `success` — 是否成功
- `master_sheet` / `pool_sheet` — 识别的 Sheet 名称
- `master_count` — 原始表商品数量（品牌和名称均非空）
- `pool_count` — 商品池商品数量
- `duplicate_count` — 一品多码商品数量
- `same_code_count` — 名称匹配但编码相同（同一商品，未标红）的数量
- `match_stats` — 匹配命中统计：`{"exact": N, "substring": N, "overlap": N}`
- `duplicates` — 一品多码详细列表，每条包含：
  - `row` — 动态商品池行号
  - `brand` — 品牌
  - `name` — 商品池中的名称
  - `pool_code` — 商品池中的编码
  - `master_code` — 原始表中匹配到的编码
  - `master_name` — 原始表中匹配到的名称
  - `match_type` — 匹配类型：`exact` 精确 / `substring` 子串包含 / `overlap` 关键词重叠

常见错误处理：

- 文件不存在：提示用户检查路径
- Sheet 数量不足：提示需要至少2个Sheet
- 表头不匹配：输出当前表头，提示期望的表头格式
- 文件被占用：提示关闭 Excel/WPS 后重试

### Step 3: 输出检查报告

根据脚本返回的 JSON 结果，向用户清晰展示：

1. 检查摘要：原始表商品数、商品池商品数、一品多码数量、同品同码数量
2. 一品多码明细（如果有）：以表格列出重复商品的行号、品牌、名称、商品池编码、原始表编码
3. 操作提示：告诉用户文件已更新，一品多码行已在动态商品池中标红

示例输出格式：

```
### SKU重复检查报告

| 项目 | 数量 |
|------|------|
| 原始表商品数 | 25,559 |
| 商品池商品数 | 81 |
| 🔴 一品多码 | 2 |
| ✅ 同品同码（跳过） | 1 |

#### 一品多码明细
| 行号 | 匹配 | 品牌 | 名称（商品池） | 商品池编码 | 原始表编码 |
|------|------|------|---------------|-----------|-----------|
| 80 | 精确 | 森耐德 | 森耐德(sennaide) 石墨盘根… | 9999999 | 8490630 |
| 82 | 模糊 | 惠普 | 惠普（HP）硒鼓 CF280A… | 8888888 | 8488992 |

已对以上2条一品多码记录在动态商品池Sheet中标红处理，请打开数据总表查看。
```

如果没有重复商品，告知用户检查通过，所有商品均为新增，无需处理。

## 数据总表结构（固定）

脚本采用固定列位置匹配，表头必须严格一致：

**Sheet1（商品原始表）：**

| 列 | 表头名称 |
|---|---------|
| A | 商品编码 |
| B | 自定义名称 |
| C | 品牌 |

**Sheet2（动态商品池）：**

| 列 | 表头名称 |
|---|---------|
| A | 名称 |
| B | 品牌 |
| C | 编码 |
| D | 单位 |

如果表头与上述不一致，脚本会报错并输出当前实际表头，方便排查。

## Dependencies

脚本依赖 `openpyxl` 库，首次使用前确保已安装：

pip install openpyxl
