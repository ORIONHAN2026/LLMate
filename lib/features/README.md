# features/ - 功能模块层

## 职责

按业务领域组织 UI 页面、组件和局部控制器，每个功能模块自包含。

## 模块列表

| 模块 | 说明 | 文件数 |
|------|------|--------|
| `chat/` | 核心聊天功能：主页、输入框、消息列表、侧边栏 | 17 |
| `mcp/` | MCP 服务管理：配置、市场、内置服务器 | 13 |
| `models/` | LLM 模型管理：详情编辑、在线添加 | 5 |
| `settings/` | 应用设置：模型/连接器/其他设置 | 3 |

## 设计原则

- **模块自治**：每个模块包含自己的 pages/、widgets/、controllers/
- **按需引用**：模块间通过 models/ 和 core/ 共享数据，不直接引用彼此的 UI
- **清晰导航**：功能查找路径 = `features/{功能名}/pages/` 或 `features/{功能名}/widgets/`
